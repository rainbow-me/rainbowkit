# स्थापना

## RainbowKit के साथ उपयोग करना शुरू करें

### त्वरित आरंभ

आप निम्नलिखित कमांडों में से किसी एक का उपयोग करके, अपने पैकेज मैनेजर की पसंद के साथ, एक नया RainbowKit + [wagmi](https://wagmi.sh) + [Next.js](https://nextjs.org) ऐप का ढांचा तैयार कर सकते हैं:

```bash
npm init @rainbow-me/rainbowkit@latest
# or
pnpm create @rainbow-me/rainbowkit@latest
# or
yarn create @rainbow-me/rainbowkit
```

यह आपसे किसी परियोजना का नाम पूछेगा, एक नई डायरेक्टरी बनाएगा जिसमें आधिकारिक परियोजना होगी, और सभी आवश्यक निर्भरताओं को स्थापित करेगा।

वैकल्पिक रूप से, आप अपने मौजूदा परियोजना में RainbowKit को मैन्युअल रूप से एकीकृत कर सकते हैं।

### मैन्युअल सेटअप

RainbowKit और इसके समकक्ष निर्भरताओं, [wagmi](https://wagmi.sh/), [viem](https://viem.sh), और [@tanstack/react-query](https://tanstack.com/query/v5) को इंस्टॉल करें।

```bash
npm install @rainbow-me/rainbowkit wagmi viem@2.x @tanstack/react-query
```

> नोट: RainbowKit एक [React](https://reactjs.org/) पुस्तकालय है।

#### आयात करें

RainbowKit, Wagmi और TanStack Query को इम्पोर्ट करें।

```tsx
import '@rainbow-me/rainbowkit/styles.css';

import {
  getDefaultConfig,
  RainbowKitProvider,
} from '@rainbow-me/rainbowkit';
import { WagmiProvider } from 'wagmi';
import {
  mainnet,
  polygon,
  optimism,
  arbitrum,
  base,
  zora,
} from 'wagmi/chains';
import {
  QueryClientProvider,
  QueryClient,
} from "@tanstack/react-query";
```

#### कॉन्फ़िगर करें

अपने वांछित चेन्स को कॉन्फ़िगर करें और आवश्यक कनेक्टर्स उत्पन्न करें। आपको `wagmi` कॉन्फ़िग सेटअप करने की भी आवश्यकता होगी। यदि आपका dApp सर्वर साइड रेंडरिंग (SSR) का उपयोग करता है, तो सुनिश्चित करें कि `ssr` को `true` पर सेट करें।

> नोट: हर dApp जो WalletConnect पर निर्भर करता है अब [WalletConnect Cloud](https://cloud.walletconnect.com/) से `projectId` प्राप्त करने की आवश्यकता है। इसमें बिल्कुल मुफ्त है और सिर्फ कुछ मिनट लगते हैं।

```tsx line=4-99
...
import { getDefaultConfig } from '@rainbow-me/rainbowkit';

const config = getDefaultConfig({
  appName: 'My RainbowKit App',
  projectId: 'YOUR_PROJECT_ID',
  chains: [mainnet, polygon, optimism, arbitrum, base, zora],
  ssr: true, // If your dApp uses server side rendering (SSR)
});
```

#### प्रदाताओं को लपेटें

अपने एप्लिकेशन को `RainbowKitProvider`, [`WagmiProvider`](https://wagmi.sh/react/api/WagmiProvider#wagmiprovider), और [`QueryClientProvider`](https://tanstack.com/query/v4/docs/framework/react/reference/QueryClientProvider) के साथ लपेटें।

```tsx
const queryClient = new QueryClient();

const App = () => {
  return (
    <WagmiProvider config={config}>
      <QueryClientProvider client={queryClient}>
        <RainbowKitProvider>
          {/* Your App */}
        </RainbowKitProvider>
      </QueryClientProvider>
    </WagmiProvider>
  );
};
```

#### कनेक्ट बटन जोड़ें

फिर, आपके ऐप में, `ConnectButton` कौम्पोनॅन्ट को आयात करें और रेंडर करें।

```tsx
import { ConnectButton } from '@rainbow-me/rainbowkit';

export const YourApp = () => {
  return <ConnectButton />;
};
```

RainbowKit अब आपके उपयोगकर्ता के वॉलेट चयन, वॉलेट/लेनदेन जानकारी प्रदर्शित करने और नेटवर्क/वॉलेट स्विच करने का प्रबंधन करेगा।

#### अतिरिक्त निर्माण उपकरण सेटअप

कुछ बिल्ड टूल्स को अतिरिक्त सेटअप की आवश्यकता होगी।

##### React, Next.js, और Webpack

Next.js और Create React App द्वारा प्रयोग किया गया [Webpack v5](https://webpack.js.org/blog/2020-10-10-webpack-5-release/) bundler [अब Node polyfills प्रदान नहीं करता](https://webpack.js.org/blog/2020-10-10-webpack-5-release/#automatic-nodejs-polyfills-removed), इसलिए आपको RainbowKit की peer dependencies को संतुष्ट करने के लिए इन मॉड्यूल्स को स्वयं शामिल करना होगा। Create React App विशेष रूप से आपको `Buffer` के लिए polyfill करने की आवश्यकता है।

RainbowKit के पिछले संस्करणों में जो [ethers](https://docs.ethers.org/v5/) पर निर्भर थे, `fs`, `net`, और `tls` मॉड्यूल्स स्वतः polyfilled होते थे। RainbowKit v1 + wagmi v1 के साथ जो [viem](https://viem.sh/) पर बनाया गया है, यह मामला अब नहीं है।

हमारे [Next.js Webpack Config](https://github.com/rainbow-me/rainbowkit/blob/main/examples/with-next/next.config.js), [Next.js App Router Webpack Config](https://github.com/rainbow-me/rainbowkit/blob/main/examples/with-next-app/next.config.js), और [Create React App polyfills](https://github.com/rainbow-me/rainbowkit/blob/main/examples/with-create-react-app/src/polyfills.ts) नमूनों की संदर्भ लें अपने प्रोजेक्ट के configuration guidance के लिए।

##### Vite

[Vite](https://vitejs.dev) bundler भी Node polyfills प्रदान नहीं करता, इसलिए आपको `global`, `Buffer` और `process.env` के लिए polyfills शामिल करने होंगे। उदाहरण के तौर पर, आप हमारे नमूना Vite प्रोजेक्ट में [polyfills का संदर्भ ले सकते हैं](https://github.com/rainbow-me/rainbowkit/tree/main/examples/with-vite/src/polyfills.ts)।

##### Remix

जब आप [Remix](https://remix.run) का उपयोग कर रहे होते हैं, तो सभी RainbowKit पैकेज प्रवेश बिंदुओं को आपके [सर्वर डिपेंडेंसीज](https://remix.run/docs/en/v1/pages/gotchas#importing-esm-packages) की सूची में आपके Remix कॉन्फ़िग में जोड़ा जाना चाहिए क्योंकि वे ESM पैकेजों के रूप में प्रकाशित किए जाते हैं।

Remix भी आपको `global`, `Buffer` और `process.env` के लिए polyfill करने की आवश्यकता है। हमारे नमूना Remix प्रोजेक्ट में [polyfills का संदर्भ लें](https://github.com/rainbow-me/rainbowkit/blob/main/examples/with-remix/app/polyfills.ts)।

```js
/**
 * @type {import('@remix-run/dev').AppConfig}
 */
module.exports = {
  serverDependencies: [
    '@rainbow-me/rainbowkit',
    '@rainbow-me/rainbowkit/wallets',
  ],
};
```

### अपनी स्वयं की कार्यक्षमता जोड़ें

अब जब आपके उपयोगकर्ता अपने बटुए को कनेक्ट कर सकते हैं, तो आप [wagmi.](https://wagmi.sh) का उपयोग करके अपने ऐप के बाकी हिस्से का निर्माण शुरू कर सकते हैं।

लेनदेन भेजें, अनुबंध के साथ बातचीत करें, ENS विवरण हल करें और अनेक अन्य चीजों के साथ wagmi के व्यापक सीट के React Hooks का उपयोग करें।

अधिक विवरण के लिए, [wagmi दस्तावेज़ीकरण](https://wagmi.sh) देखें।
