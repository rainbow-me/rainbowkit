# WalletConnect v2 में माइग्रेशन

## अपने dApp को v2 अपग्रेड के लिए 5 मिनट या उससे कम समय में तैयार करें

WalletConnect v1 प्रोटोकॉल 28 जून, 2023 को बंद कर दिया जाएगा, और RainbowKit आपके लिए ट्रांज़िशन को सुपर सरल बनाने के लिए तैयार है।

WalletConnect v2 के लिए समर्थन अब RainbowKit में मानक है, बिना किसी ब्रेकिंग परिवर्तन के। बस अपग्रेड करें और अतिरिक्त विवरणों के लिए नीचे दिए गए गाइड का संदर्भ लें।

#### WalletConnect v2 को समझना

WalletConnect वह प्रोटोकॉल है जो उपयोगकर्ताओं को अपने मोबाइल वॉलेट्स को आपके dApp से कनेक्ट करने की अनुमति देता है, और यह RainbowKit की एक महत्वपूर्ण नींव है। WalletConnect v2 की हालिया रिलीज ने वॉलेट-dApp अनुभव को और बेहतर बनाने के लिए कई महत्वपूर्ण अपग्रेड लाए हैं।

v2 के साथ, आपके उपयोगकर्ता अब एक साथ कई चेन से कनेक्ट कर सकते हैं, इसलिए आप चेन स्विचिंग हैंडशेक को भूल सकते हैं। यह अधिक नेटवर्क के लिए समर्थन को भी विस्तारित करता है, और ऐसे सुधार लाता है जो Ethereum के साथ साइन-इन को और भी तेज बनाते हैं।

#### अपने dApp को अपग्रेड करना

यदि आपने पहले ही RainbowKit `0.12.x` और `1.0.x` के लिए माइग्रेशन गाइड का पालन किया है, तो आप लगभग पहुंच गए हैं! बस नवीनतम पैच रिलीज़ में अपग्रेड करें और इसे आजमाएं।

RainbowKit ने WalletConnect v2 को डिफ़ॉल्ट रूप से `0.12.15` और `1.0.2` के साथ सक्षम किया। यदि आपका dApp अभी तक wagmi v1 और RainbowKit v1 में अपग्रेड नहीं हुआ है, तो हम `0.12.x` के लिए समर्थन जारी रख रहे हैं।

**1. निर्भरताएँ अपग्रेड करना**

```bash
npm i @rainbow-me/rainbowkit@^1.0.2 wagmi@^1.2
```

यदि आप RainbowKit के `0.12.x` या `1.0.x` से पहले के संस्करण का उपयोग कर रहे हैं, तो अपडेट प्राप्त करने के लिए [माइग्रेशन गाइड्स](/docs/migration-guide) का पालन करें।

यह अनुशंसा की जाती है कि `0.12.x` dApps wagmi v1 में अपग्रेड करना शुरू करें, क्योंकि WalletConnect v2 के स्थिरता सुधार केवल wagmi के भविष्य के संस्करणों में शामिल किए जाएंगे।

**2. एक projectId प्रदान करना**

हर dApp जो WalletConnect पर निर्भर करता है, को अब [WalletConnect Cloud](https://cloud.walletconnect.com/) से एक `projectId` प्राप्त करने की आवश्यकता है। यह पूरी तरह से मुफ्त है और केवल कुछ ही मिनटों का समय लेता है।

नीचे दिखाए गए तरीके से `getDefaultWallets` और व्यक्तिगत RainbowKit वॉलेट कनेक्टर्स को `projectId` प्रदान करें:

```ts
const projectId = 'YOUR_PROJECT_ID';
const { wallets } = getDefaultWallets({
  appName: 'My RainbowKit App',
  projectId,
  chains,
});
const connectors = connectorsForWallets([
  ...wallets,
  {
    groupName: 'Other',
    wallets: [
      readyWallet({ projectId, chains }),
      trustWallet({ projectId, chains }),
      ledgerWallet({ projectId, chains }),
    ],
  },
]);
```

RainbowKit के पास प्रकार-सुरक्षित है और जब `projectId` गायब हो तो आपको चेतावनी देगा। v2 को क्रियाशीलता में देखने के लिए हमारे उदाहरण देखें:

- [Create React App](https://codesandbox.io/p/sandbox/github/rainbow-me/rainbowkit/tree/main/examples/with-create-react-app)
- [Next.js](https://codesandbox.io/p/sandbox/github/rainbow-me/rainbowkit/tree/main/examples/with-next)
- [Next.js App Router](https://codesandbox.io/p/sandbox/github/rainbow-me/rainbowkit/tree/main/examples/with-next-app)
- [Remix](https://codesandbox.io/p/sandbox/github/rainbow-me/rainbowkit/tree/main/examples/with-remix)
- [Vite](https://codesandbox.io/p/sandbox/github/rainbow-me/rainbowkit/tree/main/examples/with-vite)
- [React Router](https://codesandbox.io/p/sandbox/github/rainbow-me/rainbowkit/tree/main/examples/with-react-router)

अतिरिक्त समर्थित विकल्पों के लिए [दस्तावेज़](https://www.rainbowkit.com/docs/custom-wallet-list#walletconnect) का संदर्भ लें।
