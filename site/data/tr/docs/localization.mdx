# Yerelleştirme

## Yerleşik çevirilerle dApp'inizin dilini özelleştirme

Varsayılan olarak, RainbowKit İngilizce dili kullanıcıları için `en-US` yerelini destekler.

Mevcutsa, RainbowKit kullanıcının [tercih ettiği dili](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/language) algılar ve uygun çevirileri seçer. Geliştiriciler varsayılan dili her zaman geçersiz kılabilir.

Kullanıcılarınız için bir dil belirtmek istiyorsanız, `RainbowKitProvider`'da `locale="zh-CN"` propunu ekleyin.

```tsx
import { RainbowKitProvider } from '@rainbow-me/rainbowkit';

export const App = () => (
  <RainbowKitProvider locale="zh-CN" {...etc}>
    {/* Your App */}
  </RainbowKitProvider>
);
```

### Next.js ile Kullanma

RainbowKit'in yerelleştirme desteği [Alt-yol Yönlendirme](https://nextjs.org/docs/pages/building-your-application/routing/internationalization#sub-path-routing) ile daha iyi çalışır.

Next.js projenizi aşağıdaki örnek gibi yapılandırın, bu da arama motorlarının ve kullanıcıların çok dilli desteğinizi daha iyi keşfetmelerine yardımcı olur.

```json
// next.config.js
{
  i18n: {
    locales: ['default', 'en', 'zh-CN'],
    defaultLocale: 'default',
  },
}
```

Daha sonra `Pages Router` tarafından sağlanan `locale`yi `RainbowKitProvider`'a iletin.

```tsx
import { RainbowKitProvider, Locale } from '@rainbow-me/rainbowkit';

export const App = () => {
  const { locale } = useRouter() as { locale: Locale };
  return (
    <RainbowKitProvider locale={locale} {...etc}>
      {/* Your App */}
    </RainbowKitProvider>
  )
};
```

Pages Router için örneği [burada](https://github.com/rainbow-me/rainbowkit/tree/main/examples/with-next) bulabilirsiniz.

App Router henüz i18n desteği sunmuyor. [`next-intl`](https://github.com/amannn/next-intl) aracı katmanıyla uygulama en iyi pratikleri için ayrı örneğimize [buradan](https://github.com/rainbow-me/rainbowkit/tree/main/examples/with-next-app-i18n) bakın.

dApp içeriğinizi tamamen yerelleştirme desteği için aynı teknikleri kullanmanız önerilir. [`i18n-js`](https://github.com/fnando/i18n) ve [`next-intl`](https://github.com/amannn/next-intl) gibi yerelleştirme kütüphaneleri ve Crowdin gibi yönetim araçları bu süreci basitleştirecektir.
